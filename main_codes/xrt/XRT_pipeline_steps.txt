#
#--------XRT pipeline
#
############################
Step 0. Initialize heasoft and caldb path

Step 1. -- Create the file with the relevant information about your source
    1a. With a text editor, open the code ***make_base_source_info_file.py** and change info about the source (RA,DEC,etc) 
    1b. Run: python make_base_source_info_file.py 
        Result: This creates the **base_info.csv** file with all the info that we need about the source for the analysis.

Step 2. -- Run xrtpipeline
    2a. Run: python xrtpipeline_run.py > xrtpipeline_run.log 
        Result: This will run the main **xrtpipeline** over **all the obsids**; the analysis results are stored in the Swift/xrtout_*obsid* folders
    2b. In the log file, check for all instances of "xrtpipeline_0.13.7: Exit" and make sure no errors are reported. Note: the pipeline version number will depend on your heasoft installation.

Step 3. -- Create the source/background region files for WT and PC
    **WT events**: 
    3a. Edits: in line 73 of **xrt_wt_make_image.py** you can change the src/bkg radius size (e.g. src_wt_size_asec = 50 ##arcsec)
    3b. Run: python xrt_wt_make_image.py  
        Result: in every Swift/xrtout_*obsid* folder, you should have created 5 files: src_wt_fk5.reg, src_wt_phys_centr.reg, src_wt_fk5_centr.reg, bkg_wt_phys, bkg_wt_fk5.reg. In the folder Swift/xrt_png/ you will find the saved WT images with the source and background regions.
        
    **PC events**: 
    3b. open with ds9 one XRT/PC event files in one Swift/xrtout_*obsid* (e.g. ds9 Swift/xrtout_00035021001/sw00035021001xpc*po_cl.evt)
    3c. select a source region (decide on the size, center on source, and centroid) --> save it as src_pc.reg in Swift/
    3d. select a bkg region (decide on the size/shape [circle or ellipse]) --> save it as bkg_pc.reg in Swift/  
    3e. Run: python xrt_pc_make_image.py > xrt_pc_make_image.log 
        Result: in every Swift/xrtout_*obsid* folder, you should have created 1 file: *obsid*_src_cent.reg. In the folder Swift/xrt_png/ you will find the saved PC images with the source and background regions.

Step 4. -- Check the images
    In the Swift/ folder, you should now see the folder xrt_png/. 
        a. Open it and check the output images to ensure your bkg and source region are well selected. 
        PC images
           b. Ensure no source is present in the background regions created. 
           c. Ensure that all src and bkg regions are inside the xrt image. 
           d. Ensure that the pointings are not defocused.  
        WT images
           e. Ensure the source and background region are on the slit. 
           
        f. If the source is at the edge of the detector, you can remove the file Swift/xrtout_*obsid*/*xpc*po_cl.evt 
        g. If the bkg image is outside some of the frames, you may want to define a different bkg region (step 3).  

Step 5. -- Run xselect to create spectra
    a. Run: python xselect_run.py > xselect_run.log 
       Result: This code will create the src.pha;bkg.pha;src_wt.pha and bkg_wt.pha  files in the various Swift/xrtout_*obsid*/ folders
    
############################
#
#--------XSPEC FIT 
#
############################

8. Copy the latest XRT RMF file you need into the Swift/ folder as xrt.rmf

9. Run: python xspec_pl_fit.py > xspec_run.log
   This code, for every xrtout_obsid does:
   a. creates an arf
   b. associates the bkg.pha, arf and rmf to the src.pha file
   c. groups using optimal binning the source spectrum 
   d. fits the spectrum with a tbabs*zpowerlaw the spectrum 
   e. Does the same for WT files 

10. Now in the Swift/ folder you should have the params_xspec.txt with all the info needed
